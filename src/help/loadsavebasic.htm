BASIC-Programme laden und speichern

Das Laden und Speichern von BASIC-Programmen erfordert eine besondere Vorgehensweise. JKCEMU bietet eine spezielle Unterstütztung für KC-BASIC-, AC1-BASIC, AC1-Mini-BASIC- und Z1013-Tiny-BASIC-Programme.

Der KC-BASIC-Interpreter meldet sich zwar mit "HC-BASIC" (Heimcomputer-BASIC), jedoch wird dieses BASIC aufgrund der später erfolgten Umbenennung der ersten DDR-Heimcomputer in KC85/1 und KC85/2 (Kleincomputer) üblicherweise KC-BASIC genannt.

Laden und Speichern von KC-BASIC-Programmen

Laden eines KC-BASIC-Programms

  1. Laden Sie den KC-BASIC-Interpreter in den Arbeitsspeicher des Emulators (nicht relevant, wenn sich der BASIC-Interpreter bereits im ROM befindet).
  2. Führen Sie einen Kaltstart aus.
  3. Verlassen Sie den BASIC-Interpreter mit BYE.
  4. Laden Sie das KC-BASIC-Programm wie eine gewöhnliche Datei in den Arbeitsspeicher (siehe Datei laden).
    Das BASIC-Programm muss dazu in einem Dateiformat gespeichert sein, welches auch als KC-BASIC erkannt wird. Das sind:
    • KC-BASIC-Programmdateien (üblicherweise *.sss-Dateien)
    • KC-TAP-Dateien, welche KC-BASIC-Programme im KC-BASIC-Format enthalten
    • KCB-Dateien (Es wird nur das reine BASIC-Programm geladen, keine weiteren Maschinencodeteile, siehe Hinweise zum KCB-Format!)
    • zusätzlich bei Z1013 mit RAM-BASIC: Headersave-Dateien mit Dateityp "B" und Startadresse 2BC0.
    Wird das BASIC-Programm aus einer Nicht-BASIC-Datei geladen, werden die Systemzellen des BASIC-Interpreters nicht angepasst, was zu unerwüschten Effekten bis hin zum Absturz des ganzen BASIC-Interpreters führen kann.
  5. Führen Sie einen Warmstart des BASIC-Interpreters aus.
  6. Jetzt können Sie das BASIC-Programm mit RUN starten.

Achtung! In der Regel ist es auch möglich, ein BASIC-Programm zu laden, während Sie sich noch im BASIC-Interpreter befinden. Allerdings darf der Interpreter gerade nichts tun, erst recht kein BASIC-Programm ausführen!

Ist das KC-BASIC-Programm in einer KCC- oder KC-TAP-System-Datei gespeichert, sodass JKCEMU diese Datei nicht als BASIC-Programm erkennt, müssen Sie folgendermaßen vorgehen: KC-BASIC-Programm aus KCC-/KC-TAP-Datei extrahieren.

Speichern eines KC-BASIC-Programms

Klicken Sie im Menü Datei, Untermenü BASIC-Programm, auf den Punkt Speichern.... Je nach dem gerade emulierten System werden Sie nun gefragt, ob das BASIC-Programm im entsprechenden Adressbereich des RAM- oder ROM-BASIC-Interpreters gesucht werden soll. Anschließend erscheint der Dialog für das Speichern einer Datei (siehe Datei speichern). Die Eingabefelder für den zu speichernden Adressbereich sind bereits ausgefüllt und können nicht geändert werden. Des weiteren ist auch das übliche Dateiformat vorausgewählt (KC-BASIC-Programmdatei bzw. bei Z1013 Headersave-Datei mit Dateityp "B").

Achtung! Sie können das BASIC-Programm prinzipiell in jedem angebotenen Dateiformat speichern. Allerdings sollten Sie ein KC-BASIC-Programm nur als KC-BASIC-Programmdatei oder als KC-TAP-Datei mit Blocknummer 1 als ersten Block speichern. Beim Z1013 ist noch das Headersave-Format mit Dateityp "B" möglich. Bei allen anderen Formaten wird das BASIC-Programm zwar auch gespeichert, allerdings werden beim späteren Laden die Systemzellen des BASIC-Interpreters nicht angepasst, und das BASIC-Programm kann dann nicht richtig genutzt werden.

Laden und Speichern von AC1-BASIC-Programmen

Dieser Rubrik bezieht sich auf BASIC-Programme für den 8 kByte großen AC1-BASIC-Interpreter Version 1.1 von 1986.

Laden eines AC1-BASIC-Programms

  1. Laden Sie den AC1-BASIC-Interpreter in den Arbeitsspeicher des Emulators.
  2. Führen Sie einen Kaltstart aus.
  3. Verlassen Sie den BASIC-Interpreter mit BYE.
  4. Laden Sie das AC1-BASIC-Programm wie eine gewöhnliche Datei in den Arbeitsspeicher (siehe Datei laden).
    Das BASIC-Programm muss in einer Headersave-Datei mit Dateityp "B" und Anfangsadresse 607F gespeichert sein. Ist das nicht der Fall, wird das BASIC-Programm zwar geladen, die Systemzellen des BASIC-Interpreters aber nicht angepasst. Das kann dann zu unerwüschten Effekten bis hin zum Absturz des ganzen BASIC-Interpreters führen.
  5. Führen Sie einen Warmstart des BASIC-Interpreters aus.
  6. Jetzt können Sie das BASIC-Programm mit RUN starten.

Achtung! In der Regel ist es auch möglich, ein BASIC-Programm zu laden, während Sie sich noch im BASIC-Interpreter befinden. Allerdings darf der Interpreter gerade nichts tun, erst recht kein BASIC-Programm ausführen!

Speichern eines AC1-BASIC-Programms

Während ein AC1 emuliert wird, klicken Sie im Menü Datei, Untermenü BASIC-Programm, auf den Punkt Speichern.... Es erscheint der Dialog für das Speichern einer Datei (siehe Datei speichern). Die Eingabefelder für den zu speichernden Adressbereich sind bereits ausgefüllt und können nicht geändert werden. Des weiteren ist auch das übliche Dateiformat vorausgewählt (Headersave-Datei mit Dateityp "B").

Achtung! Sie können das AC1-BASIC-Programm prinzipiell in jedem angebotenen Dateiformat speichern. Allerdings werden beim späteren Laden die Systemzellen des BASIC-Interpreters nur dann angepasst, wenn das BASIC-Programm in einer Headersave-Datei mit Dateityp "B" gespeichert wird. Bei allen anderen Formaten werden beim Laden die Systemzellen nicht angepasst, und das BASIC-Programm kann dann nicht richtig genutzt werden.

Laden und Speichern von AC1-Mini- und Z1013-Tiny-BASIC-Programmen

Laden eines Mini-/Tiny-BASIC-Programms

  1. Laden Sie den Tiny-BASIC-Interpreter in den Arbeitsspeicher des Emulators (nicht relevant bei AC1, da dort der Mini-BASIC-Interpreter bereits im ROM enthalten ist).
  2. Führen Sie einen Kaltstart aus.
  3. Verlassen Sie den BASIC-Interpreter mit BYE.
  4. Laden Sie das Mini-/Tiny-BASIC-Programm wie eine gewöhnliche Datei (siehe Datei laden).
  5. Führen Sie einen Warmstart des BASIC-Interpreters aus.
  6. Jetzt können Sie das BASIC-Programm mit RUN starten.


Speichern eines Mini-/Tiny-BASIC-Programms

Klicken Sie dazu im Menü Datei, Untermenü Mini-/Tiny-BASIC-Programm, auf den Punkt Speichern.... Es erscheint der Dialog für das Speichern einer Datei (siehe Datei speichern). Die Eingabefelder für den zu speichernden Adressbereich sind bereits ausgefüllt und können nicht geändert werden. Des weiteren ist auch das übliche Dateiformat vorausgewählt (Headersave-Datei mit Dateityp "b").
