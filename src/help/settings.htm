Einstellungen

Die Einstellungen nehmen Sie in einem separaten Fenster vor, dass Sie über das Menü Extra des Hauptfensters aufrufen. Die weiteren Ausführungen beziehen sich auf dieses Fenster.

Sie können Ihre Einstellungen in Form von Profilen speichern und wieder laden. Wird beim Start des Emulators der Name eines solchen Profils als Argument angegeben, startet JKCEMU gleich mit diesen Einstellungen.

1. Gruppierung der einzelnen Einstellungen

Die einzelnen Einstellmöglichkeiten sind thematisch gruppiert. Jede einzelne Gruppe finden Sie auf einer separaten Unterseite, die Sie über die Reiter am oberen Rand erreichen.

1.1. System

In diesem Bereich legen Sie das zu emulierende System, d.h. den zu emulierenden Computer-Typ, fest. Bei einigen Systemen gibt es unterschiedliche Konfigurationsmöglichenkeiten, die Sie auf der rechten Seite im Bereich Optionen beeinflussen können. Hinweise zu der Art und Weise, wie die einzelnen Computer-Typen emuliert werden, finden Sie unter Emulierte Systeme.

1.2. Geschwindigkeit

Hier legen Sie fest, welche Taktfrequenz emuliert werden soll. Standardmäßig wird die Originalgeschwindigkeit des emulierten Systems nachgeahmt. Sie können aber auch einen anderen Wert angeben oder den Emulator völlig ungebremst laufen lassen. Im letzteren Fall wird die gesamte zur Verfügung stehende Rechenleistung vom Emulator beansprucht, was dazu führen kann, dass der Emulator selbst oder auch andere Programme nur noch schwerfällig zu bedienen sind.

Wird die Geschwindigkeit auf die originale oder eine andere Taktfrequenz begrenzt, versucht der Emulator, diesen Wert zu erreichen aber nicht zu überschreiten. Sollte jedoch die zur Verfügung stehende Rechenleistung dafür nicht ausreichen, wird eine entsprechend geringere Taktfrequenz emuliert.

1.3. Zeichensatz

Hier haben Sie die Möglichkeit, eine externe Zeichensatzdatei anzugeben, die dann anstelle der im Emulator integrierten Zeichensätze verwendet wird.

Die Funktion hat jedoch nur bei den emulierten Systemen eine Wirkung, die einen separaten Zeichengenerator verwenden, d.h. Systeme ohne Vollgrafik. Bei Vollgrafiksystemen (KC85/2...4) ist der Zeichensatz im Betriebssystem enthalten und kann deshalb nur durch Austausch des Betriebssystems geändert werden. Der Austausch des Betriebssystems ist mit JKCEMU möglich, indem Sie dieses als externes ROM-Image einbinden.

1.4. ROM

Mit der Auswahl des zu emulierenden Computer-Typs sind auch die ROM-Bereiche vorgegeben. Zusätzlich können Sie diese ROM-Bereiche überschreiben bzw. neue ROM-Bereiche anlegen, indem Sie externe ROM-Images einbinden. Diese ROM-Images müssen als Speicherabbilddateien vorliegen, d.h., die Dateien werden so wie sie sind in den Arbeitsspeicher des Emulators geladen.

Ein ROM-Image binden Sie ein, indem Sie auf den Knopf Hinzufügen drücken. In dem sich nun öffnenden Dateiauswahldialog selektieren Sie die einzubindende Binärdatei. Anschließend werden Sie nach der Anfangsadresse gefragt, an der das ROM-Image im Arbeitsspeicher zu finden sein soll. Die Länge des so angelegten ROM-Bereichs ergibt sich aus der Länge der eingebundenen Datei.

ROM-Images können an jeder beliebigen Adresse eingebunden werden, egal ob das sinnvoll ist oder nicht. Sie können somit auch den Bildwiederholspeicher oder den Stack-Bereich als ROM deklarieren, was höchstwahrscheinlich nicht sinnvoll ist.

1.5. RAM-Floppies

In dieser Rubrik können Sie für jede RAM-Floppy eine Datei angeben, die beim Kaltstart des emulierten Systems automatisch in die RAM-Floppy geladen wird. Ein Kaltstart erfolgt, wenn Sie im Bereich System ein anderes emuliertes System auswählen und übernehmen oder im Hauptfenster den Menüpunkt Einschalten (Power On) auslösen.

1.6. Bildschirmausgabe

Helligkeit, Rand und alle anderen Einstellungen, die sich ausschließlich auf die Bildschirmausgabe beziehen, finden Sie in dieser Gruppe.

1.7. Bestätigungen

Hier geben Sie an, welche Aktionen erst nach der Bejahung einer Sicherheitsabfrage ausgeführt werden sollen.

1.8. Erscheinungsbild

JKCEMU ist mit der Programmiersprache Java entwickelt, der ein plattformübergreifendes Konzept zu Grunde liegt. Manche Leute wünschen, dass eine Applikation unter jedem Betriebssystem gleich aussieht, andere wiederum bevorzugen, dass sich eine Applikation an das Aussehen der jeweiligen Plattform anpasst. Java wird diesen unterschiedlichen Ansprüchen gerecht, indem es sich in verschiedenen Erscheinungsbildern präsentieren kann. JKCEMU bietet deshalb diese Möglichkeit auch an.

Welche Erscheinungsbilder auf welcher Plattform unterstützt werden, hängt von der jeweiligen Java-Version ab. In der Regel sind das aber mindestens das plattformübergreifende Erscheinungsbild Metal und ein dem lokalen Betriebssystem angepasstes Erscheinungsbild.

Die Erscheinungsbilder haben nicht nur Auswirkungen auf das Aussehen der Fenster, sondern auch auf die Funktion des Dateiauswahldialogs.

Steht nur ein einziges Erscheinungsbild zur Verfügung, d.h. wenn keine Auswahl möglich ist, wird das Unterfenster Erscheinungsbild in den Einstellungen erst gar nicht angeboten.

1.9. Sonstiges

In dieser Gruppe finden Sie die Einstellungen, die sich in keine andere Gruppe sinnvoll einordnen lassen.

1.8.1. Dateiauswahldialog

JKCEMU kommt mit einem eigenen Dateiauswahldialog daher. Dieser hat den Vorteil, dass er auf die Bedürfnisse des Emulators zugeschnitten ist. So können Sie z.B. gleich in dem Dialog wählen, ob eine Datei nur geladen oder auch gestartet werden soll. Ein anderer Vorteil ist, dass beim Anklicken einer Datei, deren Typ von JKCEMU unterstützt wird, die Kopfdaten in der Statuszeile angezeigt werden.

Nachteilig könnte sein, dass der JKCEMU-Dateiauswahldialog nicht so gut auf spezifischen Eigenheiten des Systems angepasst ist (z.B. Netzwerkumgebung). Aus diesem Grund haben Sie hier die Möglichkeit, zwischen dem JKCEMU-Dateiauswahldialog und der Dateiauswahlbox des Betriebssystems zu wählen.

Der JKCEMU-Dateiauswahldialog passt sich dem Erscheinungsbild an, und zwar funktional und optisch. Wenn sie ein Erscheinungsbild ausgewählt haben, welches dem des zugrunde liegenden Betriebs- bzw. Fenstersystems entspricht, wird sich der JKCEMU-Dateiauswahldialog möglicherweise nur wenig von dem des Betriebssystems unterscheiden.

1.8.2. Rand um Bildschirmausgabe

Sie können einen Rand um die Bildschrirmausgabe erzeugen, indem Sie das Hauptfenster größer ziehen. Wenn Sie nun jedoch eine andere Skalierung der Bildschirmausgabe wählen oder in den Einstellungen ein anderes zu emulierendes System auswählen oder ein anderes Erscheinungsbild aktivieren, passt der Emulator das Hauptfenster an die neuen Erfordernisse an, womit der Rand wieder verschwindet. Mit dieser Option können Sie einen Standardrand festlegen, der in solchen Fällen automatisch erzeugt wird.

1.8.3. Zykluszeit für Bildschirmaktualisierung

Der Emulator aktualisiert zyklisch die Bildschirmausgabe entsprechend den Änderungen im Bildwiederholspeicher bzw. im Video-RAM. Eine kürzere Zykluszeit bewirkt eine flüssigere Bildschirmausgabe. Allerdings steigt damit auch der Bedarf an Rechenleistung. Eine zu kurze Zykluszeit beeinträchtigt die Bedienung des Emulators, da dann zu wenig Rechenleistung für das Reagieren auf die Benutzeraktionen zur Verfügung steht.

Der Standardwert von 50 ms, d.h. 20 mal pro Sekunde, ist in der Regel auf den heute üblichen Arbeitsplatzrechnern ein guter Kompromiss zwischen flüssiger Bildschirmausgabe und flüssiger Bedienung.

1.8.4. Profile werden gespeichert im Verzeichnis

Hier sehen Sie zur Information den vollständigen Pfad, in dem die Profile gespeichert werden. Ist die Umgebungsvariable APPDATA gesetzt, liegt die Datei in dem Verzeichnis, welches in dieser Umgebungsvariable angegeben ist, anderenfalls im Verzeichnis für die eigenen Dateien.

2. Einstellungen übernehmen

Nachdem Sie eine Einstellung geändert haben, wird der Knopf Übernehmen aktiv, und sie können damit die neuen Einstellungen aktivieren.

Nach dem Aktivieren der neuen Einstellungen wird der Knopf Profil speichern aktiv. Damit haben Sie die Möglichkeit, die Einstellungen dauerhaft zu speichern.

3. Profile

Nach dem Übernehmen der Einstellungen können Sie diese in Form eines Profils speichern. Dazu drücken Sie auf Profil speichern.... Es erscheint ein Dialogfenster für die Auswahl bzw. Eingabe eines Profilnames. Nach dem Bestätigen werden die Einstellungen unter diesem Namen gespeichert.

Die in einem Profil gespeicherten Einstellungen können Sie auch wieder laden, indem Sie auf Profil laden... drücken. Es erscheint wieder der Profilauswahldialog. Nach dem Auswählen eines Profils und dem Bestätigen werden die Einstellungen geladen. Nun drücken Sie noch auf Übernehmen, um die geladenen Einstellungen auch zu aktivieren.

Im JKCEMU-Hauptfenster können Sie im Menü Extra durch Anwählen des Punktes Profile anwenden auch ein Profil laden und sofort aktivieren.

Eine weitere Möglichkeit des Ladens eines Profils besteht darin, den Profilnamen beim Starten des Emulators als Argument mit anzugeben. In diesem Fall werden die Einstellungen gleich beim Programmstart geladen und aktiviert.

Wird beim Programmstart kein Profilname angegeben, versucht der Emulator das Profil standard zu laden. Sie sollten deshalb Ihre am häufigsten verwendeten Einstellungen unter diesem Namen speichern.

Beim Speichern eines Profils werden auch die Positionen und Größen der gerade geöffneten Fenster gespeichert. Wenn Sie später so ein Profil anwenden, öffnen die Fenster dann wieder an den gespeicherten Positionen. Fenster, die bereits offen sind, werden davon allerdings nicht berührt.
