Speichereditor

Der Speichereditor dient zum Ansehen und Verändern der Bytes im Arbeitsspeicher des Emulators.

  • 1. Speicherbereich anzeigen
  • 2. Speicherbereich beschreiben
    • 2.1. Bytes direkt überschreiben
  • 3. Ausgewählte Bytes kopieren
  • 4. Schrift

1. Speicherbereich anzeigen

Das Fenster enthält im oberen Bereich zwei Felder zur Eingabe der Anfangs- und Endadresse. Füllen Sie diese hexadezimal aus und wählen Sie den Menüpunkt Datei → Aktualisieren an. Nun wird der entsprechende Bereich des Arbeitsspeichers in dem Fenster angezeigt.

Sie können auch aktualisieren, indem Sie auf die Enter-Taste drücken, während der Cursor im Eingabefeld für die Endadresse steht.

Achtung! Sie sehen immer den Speicherbereich, den auch der Mikroprozessor zum Zeitpunkt des Aktualisierens sieht. Wenn das emulierte System in dem Adressbereich mehrere parallel liegende Speicherbänke hat, müssen Sie selbst sicherstellen, dass die richtige Bank eingeblendet ist.

2. Speicherbereich beschreiben

Setzen Sie zuerst mit einem Mausklick den Cursor auf die Speicherstelle, die sie beschreiben möchten. Anschließend rufen Sie im Menü die Funktion Bearbeiten → Bytes überschreiben... auf. In dem sich öffnenden Dialog geben Sie die zu schreibenden Bytes ein und drücken auf OK.

Zum Positionieren des Cursors können Sie auch in eines der beiden Felder im Bereich Adresse die gwünschte Adresse eingeben und Enter drücken.

Achtung! Wenn das emulierte System in dem Adressbereich mehrere parallel liegende Speicherbänke hat, werden die Bytes in die Speicherbank geschrieben, die zum Zeitpunkt des Schreibens der Mikroprozessor gerade sieht. Sie müssen selbst sicherstellen, dass die richtige Bank eingeblendet ist.

2.1. Bytes direkt überschreiben

Neben dem oben beschriebenen Ablauf zum Beschreiben des Speicherbereichs gibt es auch die Möglichkeit, die Datenbytes direkt in der Anzeige zu ändern, indem man einfach zwei hexadezimale Ziffern eingibt und so den neuen Wert des markierten Datenbytes festlegt. Diese direkte Eingabemöglichkeit kann jedoch leicht zu einem versehentlichen Überschreiben des Speicherbereichs führen. Aus diesem Grund ist sie standardmäßig deaktiviert. Sie aktivieren diese Eingabemöglichkeit über den Menüpunkt Einstellungen → Bytes durch direkte Eingabe überschreiben. Beim Speichern eines Profils wird auch diese Einstellung auch mit gespeichert.

3. Ausgewählte Bytes kopieren

Mit dieser Funktion, die Sie im Menü Bearbeiten finden, können Sie die ausgewählten Bytes als Hex-String in die Zwischenablage kopieren und von da aus dann z.B. in den Dialog zum Suchen oder Überschreiben von Bytes wieder einfügen. Wählen Sie dazu die gewünschten Bytes aus, indem Sie die Maus mit gedrückter Maustaste ziehen oder indem Sie die Cursor-Tasten bei gedrückter Shift-Taste betätigen.

Achtung! Die zu kopierenden Bytes werden dirket aus dem Arbeitsspeicher gelesen. Wenn nun zwischenzeitlich diese Bytes geändert wurden (z.B. duch Laden von Dateien oder durch das emulierte System selbst), werden somit andere Bytes kopiert als im Speichereditor angezeigt werden.

4. Schrift

Die Schrift für die Datenanzeige kann man im Fenster JKCEMU-Einstellungen im Reiter Schriften/Symbole unter Schrift für Code-Bereiche ändern.
