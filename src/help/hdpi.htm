Hochauflösende Bildschirme

Auf hochauflösenden Bildschirmen, d.h. auf Bildschirmen mit einer Auflösung von mehr als 75 dpi (Dots Per Inch bzw. deutsch Punkte pro Zoll), sehen die JKCEMU-Fenster mitunter ziemlich klein aus. Zur Lösung des Problems bietet JKCEMU zwei Möglichkeiten an:

Die erste Möglichkeit ist, die JKCEMU-Fenster einfach größer skalieren zu lassen. Dazu wird Java 9 oder eine höhere Java-Version benötigt. Auf manchen Betriebssystemen funktioniert die Skalierung allerdings nur in 100%-Schritten.

Eine Skalierung kann mitunter etwas unschön aussehen, wenn die zu skalierenden Objekte harte Kanten haben. Das ist bei den Bildschirmausgaben der emulierten Systeme sowie bei den Symbolen (Icons) der Fall. Aus diesem Grund bietet JKCEMU noch eine andere Möglichkeit: Man kann die Schriften und Schriftgrößen frei wählen sowie bei den Symbolen zwischen zwei Größen umschalten.

Ob und welche Variante Sie wählen, hängt von Ihren Gegebenheiten und Ihrem Geschmack ab. Beide Varianten finden Sie in den Einstellungen.
